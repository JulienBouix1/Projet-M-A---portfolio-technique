Philosophie HITL

Banker-in-the-loop

L'IA ne signe pas les deals. Elle prépare, analyse, structure — et présente les options. Le banquier décide à chaque point critique, valide chaque livrable, corrige chaque erreur.

Chaque correction alimente les SKILL files et les MEMORY files. L'agent s'améliore deal après deal. C'est le vrai actif propriétaire.

Règle absolue : zéro email envoyé automatiquement. Jamais. Dans aucune itération future.

01

L'agent produit un livrable

Analyse, scoring, brouillon, valorisation — toujours traçable à une source documentée.

02

Le banquier valide ou corrige

Queue HITL dans le Cockpit. Chaque décision est loggée avec son contexte et son raisonnement.

03

La correction enrichit la mémoire

hitl_feedback_logs → SKILL files + MEMORY files. L'agent apprend les préférences spécifiques du deal et du secteur.

04

Le deal suivant démarre plus haut

Deal 1 : labourieux. Deal 10 : fluide. Deal 50 : quasi-autonome. Le compounding est l'avantage concurrentiel.
